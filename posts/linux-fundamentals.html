Linux Fundamentals for DevOps Engineers

January 13, 2025 12 min read Beginner

Why Linux for DevOps?

Linux powers most servers, cloud instances, and containers. As a DevOps engineer, Linux proficiency is essential for managing infrastructure, automating deployments, and troubleshooting systems.

Essential Linux Commands

File and Directory Operations

# Navigate directories
pwd          # Print working directory
ls -la       # List files with details
cd /path     # Change directory
mkdir dir    # Create directory
rmdir dir    # Remove empty directory
rm -rf dir   # Remove directory and contents

# File operations
touch file.txt      # Create empty file
cp source dest      # Copy file
mv old new          # Move/rename file
rm file.txt         # Delete file

File Permissions

# View permissions
ls -l file.txt

# Change permissions
chmod 755 script.sh    # rwxr-xr-x
chmod +x script.sh     # Add execute permission
chown user:group file  # Change ownership

# Permission meanings
# r (read) = 4, w (write) = 2, x (execute) = 1
# 755 = rwxr-xr-x (owner: rwx, group: r-x, others: r-x)

Process Management

# View processes
ps aux           # All running processes
top              # Real-time process monitor
htop             # Enhanced process monitor
pgrep nginx      # Find process by name

# Process control
kill PID         # Terminate process
killall nginx    # Kill all nginx processes
nohup command &  # Run command in background
jobs             # List background jobs

System Information

# System resources
free -h          # Memory usage
df -h            # Disk usage
du -sh /path     # Directory size
lscpu            # CPU information
uname -a         # System information

# Network information
ifconfig         # Network interfaces
ip addr show     # IP addresses
netstat -tulpn   # Network connections
ss -tulpn        # Socket statistics

Text Processing

# View file contents
cat file.txt     # Display entire file
less file.txt    # View file page by page
head -n 10 file  # First 10 lines
tail -f log      # Follow log file

# Search and filter
grep "pattern" file    # Search for pattern
grep -r "text" /dir    # Recursive search
find /path -name "*.log"  # Find files
locate filename        # Quick file search

# Text manipulation
sed 's/old/new/g' file    # Replace text
awk '{print $1}' file     # Print first column
sort file.txt             # Sort lines
uniq file.txt             # Remove duplicates

Package Management

Ubuntu/Debian (apt)

# Update package list
sudo apt update

# Install package
sudo apt install nginx

# Remove package
sudo apt remove nginx

# Search packages
apt search keyword

CentOS/RHEL (yum/dnf)

# Install package
sudo yum install nginx
sudo dnf install nginx

# Update system
sudo yum update
sudo dnf update

Service Management (systemd)

# Service control
sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx
sudo systemctl reload nginx

# Service status
systemctl status nginx
systemctl is-active nginx
systemctl is-enabled nginx

# Enable/disable services
sudo systemctl enable nginx   # Start on boot
sudo systemctl disable nginx  # Don't start on boot

File System and Storage

# Mount/unmount
sudo mount /dev/sdb1 /mnt
sudo umount /mnt

# Check disk usage
df -h                    # File system usage
du -sh /var/log         # Directory size
lsblk                   # List block devices

# File system check
sudo fsck /dev/sdb1

Environment Variables

# View environment variables
env
echo $PATH
echo $HOME

# Set environment variables
export VAR_NAME="value"
export PATH=$PATH:/new/path

# Persistent variables (add to ~/.bashrc)
echo 'export VAR_NAME="value"' >> ~/.bashrc
source ~/.bashrc

Shell Scripting Basics

#!/bin/bash

# Variables
NAME="DevOps"
echo "Hello $NAME"

# Conditionals
if [ -f "/etc/passwd" ]; then
    echo "File exists"
fi

# Loops
for i in {1..5}; do
    echo "Number: $i"
done

# Functions
function backup_logs() {
    tar -czf logs_$(date +%Y%m%d).tar.gz /var/log/
}

backup_logs

Log Management

# System logs
sudo journalctl -u nginx        # Service logs
sudo journalctl -f              # Follow logs
sudo journalctl --since "1 hour ago"

# Log files locations
/var/log/syslog                 # System messages
/var/log/auth.log               # Authentication
/var/log/nginx/access.log       # Nginx access
/var/log/nginx/error.log        # Nginx errors

Security Basics

# User management
sudo adduser username
sudo usermod -aG sudo username  # Add to sudo group
sudo passwd username            # Change password
su - username                   # Switch user

# SSH key management
ssh-keygen -t rsa -b 4096
ssh-copy-id user@server
ssh user@server

# Firewall (ufw)
sudo ufw enable
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw status

Performance Monitoring

# System monitoring
top              # Process monitor
htop             # Enhanced top
iotop            # I/O monitor
iftop            # Network monitor
vmstat 1         # Virtual memory stats
iostat 1         # I/O statistics

Best Practices

  • Always use sudo for administrative tasks
  • Regularly update system packages
  • Monitor system resources and logs
  • Use SSH keys instead of passwords
  • Implement proper backup strategies
  • Follow the principle of least privilege
  • Document system configurations

Conclusion

Linux mastery is crucial for DevOps success. Practice these commands regularly, understand the file system structure, and gradually explore advanced topics like kernel tuning, advanced networking, and system optimization.
