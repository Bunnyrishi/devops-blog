AWS Cloud Fundamentals for DevOps

January 11, 2025 14 min read Beginner

Introduction to AWS

Amazon Web Services (AWS) is the world's most comprehensive cloud platform, offering over 200 services including compute, storage, networking, databases, and machine learning capabilities.

Core AWS Services for DevOps

Compute Services

EC2 (Elastic Compute Cloud)

Virtual servers in the cloud:

# Launch EC2 instance using AWS CLI
aws ec2 run-instances \
    --image-id ami-0abcdef1234567890 \
    --count 1 \
    --instance-type t3.micro \
    --key-name my-key-pair \
    --security-group-ids sg-903004f8 \
    --subnet-id subnet-6e7f829e

Lambda

Serverless compute service:

# Simple Lambda function
import json

def lambda_handler(event, context):
    return {
        'statusCode': 200,
        'body': json.dumps('Hello from Lambda!')
    }

Storage Services

S3 (Simple Storage Service)

# AWS CLI S3 commands
aws s3 mb s3://my-devops-bucket
aws s3 cp file.txt s3://my-devops-bucket/
aws s3 sync ./local-folder s3://my-devops-bucket/folder/
aws s3 ls s3://my-devops-bucket/

EBS (Elastic Block Store)

# Create and attach EBS volume
aws ec2 create-volume \
    --size 10 \
    --volume-type gp3 \
    --availability-zone us-west-2a

aws ec2 attach-volume \
    --volume-id vol-1234567890abcdef0 \
    --instance-id i-1234567890abcdef0 \
    --device /dev/sdf

Networking

VPC (Virtual Private Cloud)

# Create VPC
aws ec2 create-vpc --cidr-block 10.0.0.0/16

# Create subnet
aws ec2 create-subnet \
    --vpc-id vpc-12345678 \
    --cidr-block 10.0.1.0/24 \
    --availability-zone us-west-2a

# Create Internet Gateway
aws ec2 create-internet-gateway
aws ec2 attach-internet-gateway \
    --vpc-id vpc-12345678 \
    --internet-gateway-id igw-12345678

Security Groups

# Create security group
aws ec2 create-security-group \
    --group-name web-servers \
    --description "Security group for web servers" \
    --vpc-id vpc-12345678

# Add inbound rule
aws ec2 authorize-security-group-ingress \
    --group-id sg-903004f8 \
    --protocol tcp \
    --port 80 \
    --cidr 0.0.0.0/0

Database Services

RDS (Relational Database Service)

# Create RDS instance
aws rds create-db-instance \
    --db-instance-identifier mydb \
    --db-instance-class db.t3.micro \
    --engine mysql \
    --master-username admin \
    --master-user-password mypassword \
    --allocated-storage 20

ECR (Elastic Container Registry)

# Push Docker image to ECR
aws ecr get-login-password --region us-west-2 | docker login --username AWS --password-stdin 123456789012.dkr.ecr.us-west-2.amazonaws.com

docker build -t my-app .
docker tag my-app:latest 123456789012.dkr.ecr.us-west-2.amazonaws.com/my-app:latest
docker push 123456789012.dkr.ecr.us-west-2.amazonaws.com/my-app:latest

Monitoring and Logging

CloudWatch

# Create CloudWatch alarm
aws cloudwatch put-metric-alarm \
    --alarm-name cpu-mon \
    --alarm-description "Alarm when CPU exceeds 70%" \
    --metric-name CPUUtilization \
    --namespace AWS/EC2 \
    --statistic Average \
    --period 300 \
    --threshold 70.0 \
    --comparison-operator GreaterThanThreshold

CloudTrail

Audit trail for AWS API calls:

# Create CloudTrail
aws cloudtrail create-trail \
    --name my-trail \
    --s3-bucket-name my-cloudtrail-bucket

Identity and Access Management (IAM)

IAM Policies

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject"
      ],
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}

IAM Roles

# Create IAM role for EC2
aws iam create-role \
    --role-name EC2-S3-Access \
    --assume-role-policy-document file://trust-policy.json

aws iam attach-role-policy \
    --role-name EC2-S3-Access \
    --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

Infrastructure as Code

CloudFormation Template

AWSTemplateFormatVersion: '2010-09-09'
Resources:
  MyVPC:
    Type: AWS::EC2::VPC
    Properties:
      CidrBlock: 10.0.0.0/16
      EnableDnsHostnames: true
      EnableDnsSupport: true
      
  MySubnet:
    Type: AWS::EC2::Subnet
    Properties:
      VpcId: !Ref MyVPC
      CidrBlock: 10.0.1.0/24
      AvailabilityZone: !Select [0, !GetAZs '']

DevOps Tools Integration

CodeCommit (Git Repository)

# Clone CodeCommit repository
git clone https://git-codecommit.us-west-2.amazonaws.com/v1/repos/my-repo

CodeBuild (Build Service)

# buildspec.yml
version: 0.2
phases:
  pre_build:
    commands:
      - echo Logging in to Amazon ECR...
      - aws ecr get-login-password --region $AWS_DEFAULT_REGION | docker login --username AWS --password-stdin $AWS_ACCOUNT_ID.dkr.ecr.$AWS_DEFAULT_REGION.amazonaws.com
  build:
    commands:
      - echo Build started on `date`
      - docker build -t $IMAGE_REPO_NAME:$IMAGE_TAG .
      - docker tag $IMAGE_REPO_NAME:$IMAGE_TAG $AWS_ACCOUNT_ID.dkr.ecr.$AWS_DEFAULT_REGION.amazonaws.com/$IMAGE_REPO_NAME:$IMAGE_TAG
  post_build:
    commands:
      - docker push $AWS_ACCOUNT_ID.dkr.ecr.$AWS_DEFAULT_REGION.amazonaws.com/$IMAGE_REPO_NAME:$IMAGE_TAG

Cost Optimization

  • Use appropriate instance types and sizes
  • Implement auto-scaling
  • Use Reserved Instances for predictable workloads
  • Monitor and optimize storage usage
  • Set up billing alerts
  • Use AWS Cost Explorer for analysis

Security Best Practices

  • Enable MFA for all users
  • Use IAM roles instead of access keys
  • Encrypt data at rest and in transit
  • Regularly rotate credentials
  • Monitor with CloudTrail and GuardDuty
  • Follow principle of least privilege

Conclusion

AWS provides a comprehensive platform for DevOps practices. Start with core services like EC2, S3, and VPC, then gradually explore advanced services like ECS, Lambda, and managed databases. Focus on automation, monitoring, and security from the beginning.
