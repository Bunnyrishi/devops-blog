Git & Version Control Systems: Complete Guide

January 2025 20 min read Version Control

What is Version Control?

Version control is a system that records changes to files over time so you can recall specific versions later. Git is the most popular distributed version control system used in modern software development.

Git Basics

Essential Git Commands:

# Initialize repository
git init

# Clone repository
git clone https://github.com/user/repo.git

# Check status
git status

# Add files
git add .
git add filename.txt

# Commit changes
git commit -m "Commit message"

# Push to remote
git push origin main

# Pull from remote
git pull origin main

Branching Strategies

1. Git Flow

  • main: Production-ready code
  • develop: Integration branch
  • feature/*: New features
  • release/*: Release preparation
  • hotfix/*: Critical fixes

Git Flow Commands:

# Initialize Git Flow
git flow init

# Start feature
git flow feature start new-feature

# Finish feature
git flow feature finish new-feature

# Start release
git flow release start 1.0.0

# Finish release
git flow release finish 1.0.0

2. GitHub Flow

Simpler workflow for continuous deployment:

  1. Create branch from main
  2. Make changes and commit
  3. Open pull request
  4. Review and merge
  5. Deploy from main

Advanced Git Operations

Branching and Merging:

# Create and switch to branch
git checkout -b feature-branch

# Switch branches
git checkout main

# Merge branch
git merge feature-branch

# Delete branch
git branch -d feature-branch

# Rebase
git rebase main

# Interactive rebase
git rebase -i HEAD~3

Collaboration Workflows

Pull Request Process:

  1. Fork repository (if external contributor)
  2. Create feature branch
  3. Make changes and commit
  4. Push branch to remote
  5. Create pull request
  6. Code review and discussion
  7. Merge after approval

Git Best Practices

  • Write clear, descriptive commit messages
  • Commit frequently with logical changes
  • Use .gitignore for unnecessary files
  • Keep branches focused and short-lived
  • Use signed commits for security

Git Configuration:

# Set user information
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# Set default editor
git config --global core.editor "code --wait"

# Enable GPG signing
git config --global commit.gpgsign true
git config --global user.signingkey YOUR_GPG_KEY

# Useful aliases
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit

Troubleshooting Common Issues

Common Git Commands for Fixes:

# Undo last commit (keep changes)
git reset --soft HEAD~1

# Undo last commit (discard changes)
git reset --hard HEAD~1

# Stash changes
git stash
git stash pop

# View commit history
git log --oneline --graph

# Cherry-pick commit
git cherry-pick commit-hash

# Resolve merge conflicts
git status
# Edit conflicted files
git add .
git commit
