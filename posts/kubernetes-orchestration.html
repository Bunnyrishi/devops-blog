Kubernetes Orchestration: Complete Guide

January 8, 2025 20 min read Intermediate

What is Kubernetes?

Kubernetes (K8s) is an open-source container orchestration platform that automates deployment, scaling, and management of containerized applications across clusters of hosts.

Core Kubernetes Components

Master Node Components

  • API Server: Central management entity
  • etcd: Distributed key-value store
  • Scheduler: Assigns pods to nodes
  • Controller Manager: Manages cluster state

Worker Node Components

  • kubelet: Node agent
  • kube-proxy: Network proxy
  • Container Runtime: Docker, containerd, etc.

Kubernetes Objects

Pods

Smallest deployable units containing one or more containers:

apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
spec:
  containers:
  - name: nginx
    image: nginx:1.20
    ports:
    - containerPort: 80

Deployments

Manage replica sets and rolling updates:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.20
        ports:
        - containerPort: 80

Services

Expose applications running on pods:

apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
  type: LoadBalancer

Essential kubectl Commands

# Get cluster information
kubectl cluster-info

# List all pods
kubectl get pods

# Describe a pod
kubectl describe pod nginx-pod

# Apply configuration
kubectl apply -f deployment.yaml

# Scale deployment
kubectl scale deployment nginx-deployment --replicas=5

# Get logs
kubectl logs nginx-pod

# Execute commands in pod
kubectl exec -it nginx-pod -- /bin/bash

ConfigMaps and Secrets

ConfigMaps

Store configuration data:

apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  database_url: "postgresql://localhost:5432/mydb"
  debug_mode: "true"

Secrets

Store sensitive information:

apiVersion: v1
kind: Secret
metadata:
  name: app-secret
type: Opaque
data:
  username: YWRtaW4=  # base64 encoded
  password: MWYyZDFlMmU2N2Rm  # base64 encoded

Persistent Volumes

Provide persistent storage:

apiVersion: v1
kind: PersistentVolume
metadata:
  name: pv-storage
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce
  hostPath:
    path: "/mnt/data"

Ingress Controllers

Manage external access to services:

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: nginx-ingress
spec:
  rules:
  - host: myapp.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: nginx-service
            port:
              number: 80

Monitoring and Logging

Health Checks

spec:
  containers:
  - name: app
    image: myapp:latest
    livenessProbe:
      httpGet:
        path: /health
        port: 8080
      initialDelaySeconds: 30
      periodSeconds: 10
    readinessProbe:
      httpGet:
        path: /ready
        port: 8080
      initialDelaySeconds: 5
      periodSeconds: 5

Best Practices

  • Use namespaces for resource organization
  • Implement resource limits and requests
  • Use labels and selectors effectively
  • Implement proper health checks
  • Use Helm for package management
  • Implement RBAC for security
  • Monitor cluster and application metrics

Conclusion

Kubernetes provides powerful orchestration capabilities for containerized applications. Master these core concepts and gradually explore advanced features like operators, custom resources, and service meshes to fully leverage Kubernetes in production environments.
